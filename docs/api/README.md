# Cosmos BGP Control Plane — API Reference

**API Groups:** `bgp.miloapis.com/v1alpha1` · `providers.bgp.miloapis.com/v1alpha1`
**Stability:** Alpha — fields and defaults may change without deprecation notice

---

## 1. Overview

Cosmos is the BGP control plane for the Datum.net fleet. It manages two distinct BGP planes:

| Plane | Daemon | Purpose |
|-------|--------|---------|
| **Underlay** | FRR | IPv6 unicast fabric routing between nodes and top-of-rack switches |
| **Overlay** | GoBGP | L3VPN (VPNv4/VPNv6) distribution for tenant workloads |

Each plane is driven by a separate BGP daemon instance, selected by label. The two planes run independently on the same nodes — the underlay provides reachability; the overlay distributes VPN routes over that reachable fabric.

### API Groups

| Group | Purpose |
|-------|---------|
| `providers.bgp.miloapis.com/v1alpha1` | BGP daemon lifecycle (BGPProvider) |
| `bgp.miloapis.com/v1alpha1` | BGP speaker config, sessions, advertisements, and policies |

All resources in both groups are **cluster-scoped** because BGP topology spans namespace boundaries.

---

## 2. Cluster Roles

The fleet operates three cluster roles. Each role runs a specific subset of cosmos resources.

### POP cluster

A point-of-presence cluster. Runs on every edge node.

- Runs **BGPProvider** (auto-bootstrapped by cosmos startup)
- Runs **BGPInstance** for `underlay` (FRR) and `overlay` (GoBGP)
- Receives **BGPSession** resources from the management cluster via Karmada
- Receives **BGPPeer** resources generated by the SessionReconciler
- Runs **BGPAdvertisement** for loopbacks and SRv6 locators
- Runs **BGPRoutePolicy** for import/export filtering

### Infra cluster

A regional infrastructure cluster. Hosts route reflectors.

- Runs **BGPProvider** for the RR daemon instance
- Runs **BGPInstance** for the RR speaker (VPNv4/VPNv6)
- Receives **BGPSession** resources (from the management cluster) designating RR clients
- Runs **BGPPeer** resources generated by the SessionReconciler

### Management cluster

The central orchestrator. Does not run any BGP daemons.

- Holds the authoritative **BGPExternalPeer** registry (ToR switches, upstream peers)
- Writes **BGPSession** resources and propagates them to POP and infra clusters via Karmada
- Resolves `toPeers` addresses from BGPExternalPeer or BGPProvider resources at write time

### BGPProvider

`BGPProvider` represents a single BGP daemon instance (FRR or GoBGP) running on a node. It is **auto-bootstrapped at cosmos startup** — operators do not create BGPProvider resources manually.

The provider is the anchor for all other resources: BGPInstance, BGPPeer, BGPAdvertisement, and BGPRoutePolicy all select a provider via `providerRef` or `providerSelector`.

#### Spec

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `type` | `string` | Yes | Daemon type. Enum: `FRR`, `GoBGP`. |
| `frr` | `FRRConfig` | Conditional | Required when `type` is `FRR`. Mutually exclusive with `gobgp`. |
| `gobgp` | `GoBGPConfig` | Conditional | Required when `type` is `GoBGP`. Mutually exclusive with `frr`. |

**FRRConfig**

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `endpoint` | `string` | Yes | gRPC endpoint of the FRR management daemon. Must be `localhost:<port>`. Remote endpoints are not supported. |

**GoBGPConfig**

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `endpoint` | `string` | Yes | gRPC endpoint of the GoBGP daemon. Must be `localhost:<port>`. Remote endpoints are not supported. |

#### Status

| Field | Type | Description |
|-------|------|-------------|
| `conditions` | `[]metav1.Condition` | Current state. See conditions table below. |
| `capabilities` | `ProviderCapabilities` | Address families supported, route reflection support, BFD support. |
| `resolvedEndpoint` | `string` | The gRPC endpoint that was validated and connected. |
| `daemon` | `string` | The daemon type as confirmed by the health check (`FRR` or `GoBGP`). |

**BGPProvider Conditions**

| Type | Meaning |
|------|---------|
| `Ready` | `True` when the daemon is reachable and healthy. `False` when the daemon is unreachable or returned an error on the health check. |
| `RemoteProviderNotSupported` | Set when `endpoint` is not a loopback address. Cosmos only supports local daemon connections. |

#### Example

See [`pop-bgpprovider-auto.yaml`](../examples/pop-bgpprovider-auto.yaml).

### BGPInstance

`BGPInstance` declares the BGP speaker configuration for a daemon: the local AS, router ID, address families, optional BGP timers, and best-path settings. One BGPInstance is created per plane per node (one for underlay FRR, one for overlay GoBGP).

When `routeReflector` is set, the instance configures the daemon as a route reflector for the specified cluster ID. This field is only valid in infra clusters.

#### Spec

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `providerSelector` | `metav1.LabelSelector` | Yes | Selects which BGPProvider this instance configures. |
| `asNumber` | `uint32` | Yes | Local AS number. Range: 1–4294967295. |
| `routerIDSource` | `string` | No | How the router ID is determined. `NodeIP` (default) derives it from the node's primary IPv6 address. `Manual` requires `routerID` to be set. Enum: `NodeIP`, `Manual`. |
| `routerID` | `string` | No | Router ID in dotted-decimal IPv4 notation. Required when `routerIDSource` is `Manual`. |
| `addressFamilies` | `[]AddressFamily` | No | Address families to activate. Defaults to `[{afi: IPv6, safi: Unicast}]`. |
| `timers` | `BGPTimers` | No | Session-level BGP timer defaults. |
| `bestPath` | `BestPathConfig` | No | Best-path selection parameters. |
| `routeReflector` | `RouteReflectorConfig` | No | Enables route reflector mode. **Infra clusters only.** |

**AddressFamily**

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `afi` | `string` | Yes | Address Family Indicator. Enum: `IPv4`, `IPv6`. |
| `safi` | `string` | No | Subsequent AFI. Default: `Unicast`. Enum: `Unicast`, `VPNUnicast`, `FlowSpec`. |

**BGPTimers**

| Field | Type | Description |
|-------|------|-------------|
| `holdTime` | `uint32` | Hold time in seconds. |
| `keepalive` | `uint32` | Keepalive interval in seconds. |

**BestPathConfig**

| Field | Type | Description |
|-------|------|-------------|
| `asPathMultipathRelax` | `bool` | Allow multipath across different AS paths. |

**RouteReflectorConfig**

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `clusterID` | `string` | Yes | Route reflector cluster ID in dotted-decimal IPv4 notation. |

#### Status

Status is per-provider. The `providers` field is a map keyed by BGPProvider name.

| Field | Type | Description |
|-------|------|-------------|
| `providers` | `map[string]ProviderStatus` | Per-provider reconciliation status. |

**ProviderStatus**

| Field | Type | Description |
|-------|------|-------------|
| `conditions` | `[]metav1.Condition` | Conditions for this provider. |

**BGPInstance Conditions (per provider)**

| Type | Meaning |
|------|---------|
| `Ready` | `True` when the speaker has been configured in the daemon with the spec AS, router ID, and address families. |
| `RouteReflectorInPOPCluster` | Set when `spec.routeReflector` is present on a POP cluster node. Route reflection is not supported in POP clusters. |

#### Examples

See [`pop-bgpinstance-underlay.yaml`](../examples/pop-bgpinstance-underlay.yaml) and
[`pop-bgpinstance-overlay.yaml`](../examples/pop-bgpinstance-overlay.yaml) and

### BGPPeer

`BGPPeer` represents one side of a configured BGP session within a daemon. It is **generated exclusively by the SessionReconciler** — operators do not create BGPPeer resources directly.

When a BGPSession is created, the SessionReconciler generates two BGPPeer resources: one for the local side and one for the remote side. Each BGPPeer is reconciled by the PeerReconciler, which calls `AddPeer`/`UpdatePeer`/`DeletePeer` on the daemon.

> **Do not create BGPPeer resources manually.** Creating a BGPPeer without the management labels set by the SessionReconciler will result in a warning log but the resource will not be rejected by the API. It will be reconciled, which may produce unexpected daemon state.

#### Spec

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `providerRef` | `LocalObjectReference` | Conditional | Direct reference to a BGPProvider. Mutually exclusive with `providerSelector`. |
| `providerSelector` | `metav1.LabelSelector` | Conditional | Label selector for a BGPProvider. Mutually exclusive with `providerRef`. |
| `address` | `string` | Yes | IPv6 address of the remote BGP peer. |
| `asNumber` | `uint32` | Yes | AS number of the remote peer. |
| `addressFamilies` | `[]AddressFamily` | No | Address families to negotiate. |
| `routeReflectorClient` | `bool` | No | When `true`, the daemon treats this peer as a route reflector client. |
| `allowAsIn` | `uint32` | No | Number of times the local AS is allowed in an AS path received from this peer (AS path loop relaxation). |
| `timers` | `BGPTimers` | No | Per-peer timer overrides. |
| `ebgpMultihop` | `EBGPMultihop` | No | Enables eBGP multi-hop. Mutually exclusive with `ttlSecurity`. |
| `ttlSecurity` | `TTLSecurity` | No | Enables GTSM. Mutually exclusive with `ebgpMultihop`. |

**EBGPMultihop**

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `ttl` | `uint32` | Yes | Maximum hop count. Range: 1–255. |

**TTLSecurity**

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `ttl` | `uint32` | Yes | Minimum expected TTL for incoming eBGP packets. Range: 1–255. |

#### Status

Status is per-provider. The `providers` field is a map keyed by BGPProvider name.

| Field | Type | Description |
|-------|------|-------------|
| `providers` | `map[string]ProviderStatus` | Per-provider reconciliation status. |

**BGPPeer Conditions (per provider)**

| Type | Meaning |
|------|---------|
| `Configured` | `True` when the peer has been successfully registered in the daemon via `AddPeer`/`UpdatePeer`. |
| `SessionEstablished` | `True` when the BGP FSM for this peer is in `Established` state. Refreshed on a polling interval. |

### BGPAdvertisement

`BGPAdvertisement` declares infrastructure prefixes to inject into the BGP RIB. Typical uses are loopback addresses and SRv6 locator blocks. The AdvertisementReconciler calls `AddPath` on the daemon; on deletion it calls `DeletePath`.

BGPAdvertisement targets the **underlay FRR** plane for IPv6 unicast prefixes. VPN address families are not supported on this resource.

#### Spec

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `providerRef` | `LocalObjectReference` | Conditional | Direct reference to a BGPProvider. Mutually exclusive with `providerSelector`. |
| `providerSelector` | `metav1.LabelSelector` | Conditional | Label selector for a BGPProvider. Mutually exclusive with `providerRef`. |
| `instanceRef` | `string` | Yes | Name of the BGPInstance this advertisement is associated with. |
| `prefixes` | `[]PrefixSpec` | Yes | Prefixes to advertise. Minimum 1. |
| `communities` | `[]string` | No | BGP community values in `AS:value` format (e.g. `"65000:100"`). |
| `localPref` | `*uint32` | No | LOCAL_PREF attribute value. Only meaningful for iBGP sessions. |

**PrefixSpec**

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `cidr` | `string` | Yes | The prefix to advertise. IPv6 CIDR notation. |

#### Status

Status is per-provider.

| Field | Type | Description |
|-------|------|-------------|
| `providers` | `map[string]ProviderStatus` | Per-provider reconciliation status. |

**BGPAdvertisement Conditions (per provider)**

| Type | Meaning |
|------|---------|
| `Advertised` | `True` when all prefixes have been injected into the daemon RIB. |
| `UnsupportedAddressFamily` | Set when a prefix or the instance's address family is not supported by this resource (e.g., VPN SAFI). |

### BGPRoutePolicy

`BGPRoutePolicy` declares import or export filtering rules for a BGP instance. The RoutePolicyReconciler applies defined sets and policies via `AddDefinedSet`/`AddPolicy` on the daemon. Statements are evaluated in order; the first match wins.

#### Spec

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `providerRef` | `LocalObjectReference` | Conditional | Direct reference to a BGPProvider. Mutually exclusive with `providerSelector`. |
| `providerSelector` | `metav1.LabelSelector` | Conditional | Label selector for a BGPProvider. Mutually exclusive with `providerRef`. |
| `instanceRef` | `string` | Yes | Name of the BGPInstance this policy applies to. |
| `direction` | `string` | Yes | Policy direction. Enum: `Import`, `Export`. |
| `statements` | `[]PolicyStatement` | Yes | Ordered policy rules. Minimum 1 item. |

**PolicyStatement**

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `prefixSet` | `[]PrefixMatch` | No | Prefixes to match. If empty, the statement matches all prefixes. |
| `communitySet` | `[]string` | No | Community values to match. |
| `action` | `string` | Yes | Action on match. Enum: `Accept`, `Reject`. |

**PrefixMatch**

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `cidr` | `string` | Yes | Prefix to match. |
| `maskLengthMin` | `*uint32` | No | Minimum prefix length (inclusive). |
| `maskLengthMax` | `*uint32` | No | Maximum prefix length (inclusive). |

#### Status

Status is per-provider.

| Field | Type | Description |
|-------|------|-------------|
| `providers` | `map[string]ProviderStatus` | Per-provider reconciliation status. |

**BGPRoutePolicy Conditions (per provider)**

| Type | Meaning |
|------|---------|

### BGPSession

`BGPSession` declares a bilateral BGP session intent: who is peering with whom, over which address families, with what parameters. BGPSession resources are written by the management cluster cosmos and propagated to POP and infra clusters via Karmada.

The SessionReconciler reads BGPSession and generates two BGPPeer resources — one for the local side (matched by `fromProviderSelector`) and one for the remote side. **BGPPeer is the only path to a configured BGP peer in the daemon.**

The `toPeers` list is resolved at write time by the management cluster:
- For ToR and external peers: from the BGPExternalPeer registry
- For infra RR peers: from BGPProvider resources in the infra cluster

#### Spec

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `fromProviderSelector` | `metav1.LabelSelector` | Yes | Selects the local BGPProvider(s). Used to determine which cluster and node owns the local side. |
| `fromInstanceRef` | `string` | Yes | Name of the local BGPInstance. |
| `toPeers` | `[]SessionPeer` | Yes | List of remote peers. Minimum 1. |
| `addressFamilies` | `[]AddressFamily` | Yes | Address families to negotiate for this session. |
| `allowAsIn` | `uint32` | No | AS path loop relaxation for all peers in this session. |
| `timers` | `BGPTimers` | No | Timer defaults for all peers in this session. |

**SessionPeer**

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `address` | `string` | Yes | IPv6 address of the remote peer. Resolved at write time. |
| `asNumber` | `uint32` | Yes | AS number of the remote peer. |
| `instanceRef` | `string` | Yes | Name of the BGPInstance on the remote side. |
| `routeReflectorClient` | `bool` | No | When `true`, the local daemon treats this peer as an RR client. Used in infra cluster RR sessions. |

#### Status

| Field | Type | Description |
|-------|------|-------------|
| `conditions` | `[]metav1.Condition` | Flat status for the session object. |

**BGPSession Conditions**

| Type | Meaning |
|------|---------|
| `Reconciled` | `True` when all BGPPeer resources for this session have been generated. |
| `DeletionBlocked` | Set when deletion is requested but the session has active BGPPeer dependents that have not yet been cleaned up. |

#### Examples

See [`mgmt-bgpsession-underlay-tor.yaml`](../examples/mgmt-bgpsession-underlay-tor.yaml),
[`mgmt-bgpsession-overlay-rrs.yaml`](../examples/mgmt-bgpsession-overlay-rrs.yaml), and
[`infra-bgpsession-rr-client.yaml`](../examples/infra-bgpsession-rr-client.yaml).

### BGPExternalPeer

`BGPExternalPeer` is the registry of BGP peers that exist outside the cosmos fleet: top-of-rack switches, upstream transit routers, and other external devices. Resources live in the management cluster only and are never propagated to POP or infra clusters.

The management cluster cosmos reads BGPExternalPeer resources when constructing BGPSession `toPeers` lists.

| `asNumber` | `uint32` | Yes | AS number of the external peer. |

#### Status

| Field | Type | Description |

**BGPExternalPeer Conditions**

## 4. Design Principles

**Provider abstraction.** All reconciliation is mediated through BGPProvider. Controllers never talk to a daemon directly — they reconcile through the provider API. This allows FRR and GoBGP to be driven by the same controller logic with different backends.

**BGPSession is the only path to BGPPeer.** No component other than the SessionReconciler may create BGPPeer resources. This ensures that all BGP sessions have corresponding session intent recorded in the management cluster.

**Labels drive dispatch.** Controllers select providers, instances, and peers using `metav1.LabelSelector`. This makes topology configuration declarative and allows dynamic reassignment by changing labels.

**Per-provider status.** BGPInstance, BGPPeer, BGPAdvertisement, and BGPRoutePolicy each carry a `providers` map in their status. If a selector matches multiple providers (e.g., on a multi-daemon node), each provider's reconciliation state is tracked independently.

**Management cluster as source of truth.** BGPSession and BGPExternalPeer live authoritatively in the management cluster. Karmada propagates BGPSession resources to target clusters based on `fromProviderSelector` labels. The management cluster never runs BGP daemons.

**Daemon lifecycle separation.** The BGPProvider auto-bootstrap at cosmos startup is separate from the reconciliation controllers. Providers are always present before any BGPInstance or BGPPeer reconciliation begins.

## 5. Operational Contract — GoBGP API Ownership

When a CNI plugin and cosmos both interact with the GoBGP daemon on the same node, the following ownership boundaries apply. Violating these boundaries will cause reconciliation conflicts or session disruption.

| GoBGP API surface | Owner |
|---|---|
| `StartBgp`, speaker config | Cosmos exclusively |
| `AddPeer` / `UpdatePeer` / `DeletePeer` | Cosmos exclusively |
| `AddPolicy` / `AddDefinedSet` and all route policy | Cosmos exclusively |
| `AddVrf` / `DeleteVrf` | CNI exclusively |
| `AddPath` / `DeletePath` | CNI exclusively |
| `ListPeer` / `ListPath` (read-only) | Either |

Cosmos does not manage VRF instances or tenant path injection. The CNI plugin manages VRF lifecycle independently of cosmos.

## 6. Deletion Semantics

All resources in both API groups carry the finalizer:

```
cosmos.bgp.miloapis.com/cleanup
```

The cleanup finalizer ensures that daemon state is removed before the Kubernetes object is deleted. The reconciler removes daemon state (peers, policies, advertisements), then removes the finalizer.

A resource stuck in `Terminating` state means the cleanup finalizer has not been removed. This can happen when the controller is not running, the daemon is unreachable, or a dependency check is blocking deletion.

**To force-remove a stuck resource (destructive):**

```bash
kubectl patch <kind>/<name> --type=merge -p '{"metadata":{"finalizers":[]}}'
```

> **Warning:** Forcing finalizer removal bypasses daemon cleanup. Any peers, policies, or routes the resource managed will remain in the daemon until the next full reconciliation or daemon restart. Use this only when the daemon state is already known to be gone (e.g., daemon was rebuilt) or when you are intentionally abandoning orphaned daemon state.

### Dependency-blocked deletion

Some resources block deletion until their dependents are removed:

- **BGPExternalPeer**: deletion is blocked if any BGPSession references this peer in its `toPeers` list. Remove or update the BGPSession first.
- **BGPProvider**: deletion is blocked if any BGPInstance, BGPPeer, BGPAdvertisement, or BGPRoutePolicy still selects this provider. Remove those resources first.

The `DeletionBlocked` condition is set on the resource to indicate which dependency is preventing deletion.

## 7. Status Model

Resources use one of two status layouts depending on their scope.

### Per-provider status

Resources that are reconciled against one or more daemon instances carry a `providers` map in their status. The key is the BGPProvider name; the value contains a `conditions` array for that provider.

This applies to: **BGPInstance**, **BGPPeer**, **BGPAdvertisement**, **BGPRoutePolicy**.

```yaml
status:
  providers:
    node-1-frr:
      conditions:
        - type: Ready
          status: "True"
          reason: Configured
          message: "Speaker configured with AS 65000"
          lastTransitionTime: "2024-01-01T00:00:00Z"
    node-2-frr:
      conditions:
        - type: Ready
          status: "False"
          reason: DaemonUnreachable
          message: "gRPC dial to localhost:50051 failed"
          lastTransitionTime: "2024-01-01T01:00:00Z"
```

If `providerRef` is used instead of `providerSelector`, the map will contain exactly one entry.

### Flat status

Resources that are not reconciled against a daemon, or that represent single objects, carry a flat `conditions` array.

This applies to: **BGPProvider**, **BGPSession**, **BGPExternalPeer**.

```yaml
status:
  conditions:
    - type: Ready
      status: "True"
      reason: DaemonResponsive
      message: "FRR daemon is reachable"
      lastTransitionTime: "2024-01-01T00:00:00Z"
```

## 8. Metrics

Operational counters for BGP sessions are exposed as Prometheus metrics rather than CRD status fields. This avoids high-frequency status writes and eliminates multi-writer races in DaemonSet deployments.

| Metric | Labels | Description |
|--------|--------|-------------|
| `bgp_session_state` | `session`, `state` | 1 for the active FSM state, 0 for all others. |
| `bgp_received_prefixes_total` | `session` | Prefixes received from the remote peer. |
| `bgp_advertised_prefixes_total` | `advertisement` | Prefixes advertised per BGPAdvertisement. |
| `bgp_session_flaps_total` | `session` | Times this session left Established state. |

Metrics are scraped from each cosmos controller pod on `:8082/metrics`.

## 9. Common Verification Commands

```bash
# List all providers and their ready status
kubectl get bgpproviders

# Check BGPInstance reconciliation status
kubectl get bgpinstances -o wide

# List all sessions and their resolved BGPPeer status
kubectl get bgpsessions
kubectl get bgppeers

# Watch for BGPPeer condition changes
kubectl get bgppeers -w

# Check whether a specific peer is established (per-provider status)
kubectl get bgppeer <name> \
  -o jsonpath='{.status.providers.<provider-name>.conditions[?(@.type=="SessionEstablished")].status}'

# Force-remove a stuck finalizer (destructive — see deletion semantics)
kubectl patch bgpinstance/underlay --type=merge -p '{"metadata":{"finalizers":[]}}'
```
